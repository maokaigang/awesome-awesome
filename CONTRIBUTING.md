# Contributing to awesome-awesome

Thanks for helping improve this list.

## What to submit

Submit repositories that are:

- Primarily an `awesome-xxx` list, index, or high-signal curated hub.
- Public and accessible without sign-in.
- Valuable for broad audiences in their topic.

## What not to submit

- Personal promo repos with weak curation.
- Abandoned lists with stale or broken content.
- Duplicates already listed in this repo.

## Submission checklist

- Add the repo in exactly one category file under `categories/`.
- Keep entries sorted alphabetically by `owner/repo`.
- Use one-line neutral descriptions.
- Prefer canonical GitHub URLs (no tracking params).
- Ensure the link is reachable.

## Pull request title

Use this format:

```text
add: owner/repo to <category>
```

## Optional quality signals

When deciding whether to include a repo, consider:

- Recent commit activity.
- Clear maintenance ownership.
- Good issue/PR hygiene.
- Practical depth over list length.
