# 为 awesome-awesome 做贡献

感谢你帮助改进这份清单。

## 可以提交什么

请提交满足以下条件的仓库：

- 主要是 `awesome-xxx` 清单、索引，或高质量策展资源库。
- 仓库名以 `awesome` 开头（大小写不敏感）。
- 公开可访问，无需登录。
- 对所属主题下的广泛用户有实际价值。

## 不要提交什么

- 策展质量较弱的个人宣传仓库。
- 内容陈旧或大量失效链接的废弃清单。
- 本仓库里已经收录的重复项。

## 提交检查清单

- 只在 `categories/` 下的一个分类文件中添加该仓库。
- 保持条目按 `owner/repo` 字母序排序。
- 描述保持单行、中性、简洁。
- 优先使用规范的 GitHub 链接（不带跟踪参数）。
- 确保链接可访问。
- 若候选仓库 `Star >= 5,000`，即使评分低于默认阈值，也可提交（前提是不触发排除规则）。
- 默认优先非 fork 仓库；若 fork 的 Star 高于上游/source，则保留 Star 更高者。
- 不允许同时收录 fork 与其上游/source；同一体系只能保留一个。

## Pull Request 标题

请使用以下格式：

```text
add: owner/repo to <category>
```

## 可选质量信号

判断是否收录时，可参考：

- 最近提交活跃度。
- 维护者归属是否清晰。
- issue/PR 维护卫生情况是否良好。
- 是否注重实用深度，而非单纯追求数量。
