# Curation Policy

This document defines how entries are selected for `awesome-awesome`.

## Evaluation Rubric

Score each candidate from 1 to 5 on each dimension:

- Scope clarity: Is the list topic clear and focused?
- Practical value: Does it help real implementation work?
- Freshness: Is maintenance active in recent months?
- Quality control: Are links/descriptions organized and usable?
- Trust signal: Is it known in the ecosystem or used by practitioners?

Recommended inclusion threshold: total score >= 18/25.

## Exclusion Heuristics

Do not include repos with:

- Mostly dead links or outdated sections.
- Very low signal and obvious SEO/spam behavior.
- No maintenance history for long periods with broken core content.

## Maintenance Cadence

- Weekly: triage incoming suggestions.
- Monthly: run link checks and prune dead entries.
- Quarterly: review category structure and split/merge if needed.
